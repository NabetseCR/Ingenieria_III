### Pruebas funcionales

Garantia de calidad de software, grado con el que un sistema o proceso cumple espectativas de un cliente o usuario.

Concordancia con los requerimientos establecidos, ademas que el usuario pueda utilizar el software con un nivel de calidad garantizado.

Presente en herrramientas de analisis, disenno, programacion.


## Testing

Actividad desarrollada para evaluar la calidad del software, para ubicar defectos y oportunidades de mejoras en el software.


Consiste en la verificacion dinamica del comportamiento del software, buscando un resultado esperado.

Esto puede ser dado de un conjunto infinito de casos.


*Verificacion dinamica* Se hace la ejecucion haciendo valores que pueden cambiar.


*Casos de pruebas seleccionados y comportamiento esperado*

Se busca una forma en el que el software funcione, en un filtro de casos seleccionados.

*Criterio de aceptacion*

Caracteristicas que un producto debe cumplir.

No deben tener ambiguedad, ser interpretados de la misma forma por cualquier persona que los lea, seguir tambien atomicidad, solo se puede cumplir o no, deben ser tambien verificables y completos.


Se definen en ellos como debe comportarse en la aplicacion para que una determinada accion se lleve a cabo.


Definir con Como, que, quien

*Caso de prueba*

Conjunto de valores de entrada, precondiciones, resultados esperados y postcondiciones de ejecucion en un requisito especifico, estos deben de estar previamente refinados con el product owner.


Estos casos al ejecutarlos puede brindar resultados como un fallo o exito.

*Incidentes*

Ocurrencia o eventu durante la ejecucion de las pruebas y requiere investigacion y eventual solucion.

Se debe registrar correctamente, mediante capturas de pantalla, codigo error y ademas si es necesario estado de base de datos.
